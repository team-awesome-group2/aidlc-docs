# 用户角色画像

## 角色 1：员工（Employee）

| 属性 | 描述 |
|------|------|
| **姓名** | 李明 |
| **角色标识** | employee |
| **描述** | 企业内部员工，通过积分商城兑换商品作为福利 |
| **目标** | 浏览商品、使用积分兑换心仪商品、查看兑换记录和积分明细 |
| **痛点** | 希望兑换流程简单快捷，能清楚看到积分余额和消费记录 |
| **技术水平** | 中等，熟悉基本 Web 操作 |
| **权限范围** | 仅限员工端功能，不能访问管理后台，不能兑换商品以外的操作 |

## 角色 2：管理员（Admin）

| 属性 | 描述 |
|------|------|
| **姓名** | 王芳 |
| **角色标识** | admin |
| **描述** | 商城运营管理员，负责商品上架、积分发放、订单处理等后台运营 |
| **目标** | 高效管理商品目录、及时处理订单、准确发放积分、掌握运营数据 |
| **痛点** | 需要清晰的数据仪表盘辅助决策，批量操作提升效率 |
| **技术水平** | 较高，熟悉后台管理系统操作 |
| **权限范围** | 仅限管理端功能，不能兑换商品 |

## 角色 3：系统（System）

| 属性 | 描述 |
|------|------|
| **姓名** | AWSome Shop 系统 |
| **角色标识** | system |
| **描述** | 系统内部自动化角色，代表服务间通信和自动化流程 |
| **目标** | 保证跨服务数据一致性、Token 验证、补偿事务执行 |
| **痛点** | 网络故障、服务不可用时需要可靠的补偿机制 |
| **权限范围** | 内部接口，不对外暴露 |

## 角色权限矩阵

| 角色 | 可访问员工端 | 可访问管理端 | 可兑换商品 |
|------|------------|------------|----------|
| 员工 | ✅ | ❌ | ✅ |
| 管理员 | ❌ | ✅ | ❌ |
| 系统 | 不适用 | 不适用 | 不适用 |
